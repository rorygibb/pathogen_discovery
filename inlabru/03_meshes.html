Meshes

Basic mesh building

Load a version of the gorilla data with extracted points gonests and boundary goboundary

library(inlabru)
library(INLA)
init.tutorial()

data("gorillas", package = "inlabru")
gonests <- gorillas$nests
goboundary <- gorillas$boundary

Let's plot the data and the surrounding domain boundary

ggplot() + gg(gonests) + gg(goboundary) + coord_fixed(ratio = 1)

Use the boundary information to build a basic mesh of the domain

mesh0 <- inla.mesh.2d(boundary=goboundary, max.edge=0.25)
ggplot() + gg(gonests) + gg(mesh0) + gg(goboundary) + coord_fixed(ratio = 1)

We know that there are boundary effects, but how large are they? We can move the computational boundary outside the domain of interest, by an amount related to the spatial correletion length

mesh1 <- inla.mesh.2d(boundary=goboundary, max.edge=c(0.25, 1))
ggplot() + gg(gonests) + gg(mesh1) + gg(goboundary) + coord_fixed(ratio = 1)

Interactive mesh building and quality assessment

We can build some intuition by exploring different mesh settings and correlation lengths. Use meshbuilder() from the experimental INLA package to investigate how the offset and max.edge parameters affect the accuracy of the SPDE approximation. Activate the assessment calculations and choose “Approximate SD deviation”. In a good mesh, the SD deviation is close to \(1\) in the domain of interest for true correlation range. Unfortunately, we don't yet know what the true correlation length is, but we can explore some plausible scenarios.

Basic mesh building

  • Start the interactive app with meshbuilder()

  • The app starts on the Inputs tab, and provides some random points to build a mesh around. Drag the offset sliders and observe the effect on the automatically generated boundary curves indicating the inner and out boundary.

  • When you're happy with the shapes, select the Display tab to se the generated mesh. You can keep modifying the offset sliders, but now you should also vary the max.edge values.

  • Below the mesh plot on the Display tab, some information about the mesh is shown; the number of vertices, the shortest edge length, and the largest edgelength.

  • You can see the code you would need run to generate the displayed mesh by switching to the Code tab.

Basic mesh assessment

  • On the Display tab, click Active under Mesh resolution assessment. This switches on some basic mesh quality assessment displays, also shown underneath the plot.

  • We want to construct a mesh that has good approximation properties in its inner part, but may have less good properties in the outer part. You will now also need to experiment with the correlation range to achieve this (and optionally the Matérn smoothness \(\nu\)).

  • The standard deviations (SD) for the exact model increase towards the domain boundary. Make sure your outer domain is wide enough to move this effect away from the domain of interest.

  • The correlation function (Correlation) for a reference point (click the mesh plot to select a reference point) may also be helpful and/or interesting.

  • The Approximate SD deviation is close to \(1\) (white) where the triangles are small enough to reliably reproduce the covariance structure of the model, and larger/smaller than \(1\) (red/blue) where the triangles are too large. This effect is seen when the correlation range is small enough compared with the triangles that the piecewise linear model on the mesh cannot capture a significant amount of small scale variability.

  • SD bound is the largest possible standard deviation at each location, that would be obtained if the field values were perfectly correlated. This can make it easier to see the extent of the SD boundary effect.

Advanced mesh construction

  • Go back to the Inputs tab.

  • Build a mesh using goboundary as inner boundary, removing RANDOM as the Boundary domain points. A simple automatic outer boundary will be constructed that is not visible in the input tab.

  • If you like, you can use gonests to generate a smooth outer boundary (which will also be visible in the input tab).

  • If you have chosen to include gonests, you should move the offset sliders a bit so that both generated boundaries are outside goboundary. The scale for the sliders increases/decreases as you increase/decrease the values. This can sometimes take a while to adjust.

  • Before leaving the tab, make sure you increase or decrease the offset and max.edge slider values to reasonable values, so that the mesh construction doesn't become too expensive! Values between 0.1 and 1, might be reasonable starting points for max.edge in this particular case, and above 1 for offset.

  • Now go back to the Display tab and repeat the basic assessment exercise

Advanced mesh assessment (extra) – only do this if you have lots of time left

  • If your computer isn't too slow, you can turn on Advanced assessment in addition to 'Active'. This will generate a Fine mesh of high resolution that is computationally expensive to handle, but which provides a closer approximation to the continuous domain model.

  • SD and Correlation for the Fine mesh are close to the true model values.

  • SD deviation shows the ratio between the Mesh resolution standard deviations and the Fine counterparts.

  • For correlations, you can inspect the differences between Mesh resolution correlations and Fine resolution correlations.
